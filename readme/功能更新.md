## 2026-05-02 - v1.8.0 新增互斥播放系统 + AlphaTab v1.8.2 升级

### 🎵 新增功能：谱例互斥播放

**功能描述**：
当用户点击播放任意一个谱例时，**自动停止所有其他正在播放的谱例**，确保同一时间只有一个谱例在发声。

**使用场景**：
- ✅ 用户在教程页面切换不同练习曲目
- ✅ 用户在 songs.html 页面对比不同版本的歌曲
- ✅ 避免多个音频同时播放造成混乱和干扰

**技术实现**：

```javascript
/* 在 togglePlay(id) 函数中，播放新谱例前执行： */

/*
 * ====== 互斥播放：停止所有其他正在播放的谱例 ======
 * 实现原理：
 * 1. 遍历 AppState.players 中所有谱例
 * 2. 找到所有正在播放且不是当前谱例的（otherId !== id）
 * 3. 调用 otherPlayer.api.stop() 停止它们
 * 4. 更新它们的按钮状态为"▶ 播放"
 */
var allPlayerIds = Object.keys(AppState.players);

for (var i = 0; i < allPlayerIds.length; i++) {
    var otherId = allPlayerIds[i];
    if (otherId === id) continue; /* 跳过当前谱例 */
    
    var otherPlayer = AppState.players[otherId];
    
    if (otherPlayer && otherPlayer.playing) {
        /* 停止并更新UI */
        otherPlayer.api.stop();
        otherPlayer.playing = false;
        
        var btn = document.getElementById('btnPlay-' + otherId);
        if (btn) {
            btn.textContent = '▶ 播放';
            btn.classList.remove('playing');
        }
    }
}
```

**用户体验提升**：

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 播放小星星后点生日歌 | ❌ 两个同时发声（混乱） | ✅ 小星星自动停止，只播生日歌 |
| 在 songs.html 切换歌曲 | ❌ 需手动停止前一首 | ✅ 自动停止，无缝切换 |
| 多个教程谱例 | ❌ 可能多个背景音重叠 | ✅ 始终只有一个在播放 |

**调试日志**：
控制台会输出 `[互斥]` 标记的详细日志：
```
[互斥] 🛑 开始检查并停止其他正在播放的谱例...
[互斥] 🛑 正在停止谱例: twinkle
[互斥] ✅ 已停止谱例: twinkle
[互斥] ✅ 共停止了 1 个正在播放的谱例
```

### 🐛 关键 Bug 修复（延续 v1.6.3）

**问题现象**:
- 页面加载时控制台报错：`Error AT202(3,33)->(3,34): Unexpected 'Pipe' token`
- 仅影响 **5.1 小星星（Twinkle）** 和 **5.3 生日快乐歌（Happy Birthday）**
- 其他 13 个 Score 正常工作

**错误信息详情**:
```
[AlphaTab][AlphaTex] Error while parsing alphaTex: There are errors in the parsed alphaTex
Error AT202(3,33)->(3,34): Unexpected 'Pipe' token. Expected one of following: Number,String,Ident
```

### 🔍 根因分析

| 分析步骤 | 结论 |
|----------|------|
| **错误位置** | 第 3 行第 33 个字符 = 小节线 `\|` |
| **触发条件** | `:duration` 时值标记紧接在 `\|` 前面 |
| **问题代码** | `'... 0.3 0.3 0.2:2 \| ...'` （❌ 错误） |
| **正常代码** | `'... 0.2 0.2 0.1:2'` （✅ 行末无 `\|`） |

**AlphaTab v1.8.x 语法规则**:
```alphaTex
# ❌ 错误：:duration 在 | 前面（解析器混淆）
'0.3 0.3 0.2:2 | 0.1 0.1 ...'
              ^^^^ 第31-33位
                    ↑ 第34位：Unexpected 'Pipe' token

# ✅ 正确方案1：使用 .duration 格式（Beat Duration）
'0.3 0.3 0.2.2 | 0.1 0.1 ...'
              ^^^^ Beat Duration，不与 | 冲突

# ✅ 正确方案2：将 :duration 放在行末
'0.1 0.1 0.2 0.2 | 0.3 0.3 0.2 | 0.1 0.1 0.3 0.3 0.1:2'
                                                    ^^^^ 行末，后面无 |
```

**官方文档参考**: [AlphaTab Document Structure](https://alphatab.net/docs/alphatex/document-structure)

### 🛠️ 修复方案

**采用 Beat Duration 格式（`.duration`）替代 Duration Change（`:duration`）**

#### 修改的 Score 数据：

| Score ID | 名称 | 修改前 | 修改后 | 小节数 |
|----------|------|--------|--------|--------|
| `twinkle` | 小星星 | `0.2:2 \|` (❌) | `0.2.2` (✅) | 4 → 4 小节 |
| `birthday` | 生日歌 | `0.2:2 \|` (❌) | `0.2.2` (✅) | 4 → 4 小节 |

**具体修改示例（twinkle 小星星）**:

**修改前（错误）**:
```javascript
'twinkle': [
    '\\instrument "Distortion Guitar"',
    '\\tempo 80',
    '.',
    '0.1 0.1 0.2 0.2 | 0.3 0.3 0.2:2 | 0.1 0.1 0.3 0.3 0.2 0.2 0.1:2',  // ❌ :2 在 | 前
    '0.1 0.1 0.2 0.2 | 0.3 0.3 0.2:2 | 0.1 0.1 0.3 0.3 0.2 0.2 0.1:2'   // ❌ :2 在 | 前
].join('\n')
```

**修改后（正确）**:
```javascript
'twinkle': [
    '\\instrument "Distortion Guitar"',
    '\\tempo 80',
    '.',
    '0.1 0.1 0.2 0.2 | 0.3 0.3 0.2.2',        // ✅ 使用 .2 格式
    '0.1 0.1 0.3 0.3 0.2 0.2 0.1.2',           // ✅ 使用 .2 格式
    '0.1 0.1 0.2 0.2 | 0.3 0.3 0.2.2',        // ✅ 使用 .2 格式
    '0.1 0.1 0.3 0.3 0.2 0.2 0.1.2'            // ✅ 使用 .2 格式
].join('\n')
```

| 时值格式 | 语法 | 位置 | 用途 | 注意事项 |
|----------|------|------|------|----------|

## 2026-05-02 - v1.4.1 修复 GTP 曲目播放问题

1. **修复 GTP 曲目无法播放的问题**
   - **问题现象**：songs.html 页面的 GTP 格式曲目点击播放无反应
   - **根本原因**：`initGtpTab()` 函数中的文件加载方式不正确
     - 原代码：手动使用 `fetch()` 获取文件数据，然后调用 `api.load(new Uint8Array(data), [0])`
     - 问题：AlphaTab v1.3.1 的 `api.load()` 方法在浏览器环境中应直接接受 URL 参数
   - **修复方案**：

     ```javascript
     // 修改前（错误）
     fetch(gtpInfo.file)
         .then(response => response.arrayBuffer())
         .then(data => api.load(new Uint8Array(data), [0]));
     
     // 修改后（正确）
     api.load(gtpInfo.file);  // 直接传递文件URL
     ```
   - **技术说明**：
     - AlphaTab v1.3.1 的 `api.load(url)` 方法会自动处理：
       1. 通过内部 fetch 下载文件
       2. 自动识别文件格式（.gp3/.gp4/.gp5/.gpx/.gp）
       3. 解析乐谱数据并渲染到容器
     - 这种方式更简洁、更可靠，避免了手动处理二进制数据的复杂性

## 2026-05-01 - v1.2.0 AlphaTex 格式完善

### 功能增强

1. **添加音符时值标记**
   - 问题：所有音符默认为四分音符，无法区分八分音符、二分音符等
   - 修复：在 Scores 中添加时值标记
   - 语法说明：
     - `:4` 或默认 = 四分音符（一拍）
     - `:8` = 八分音符（半拍）

     - `:1` = 全音符（四拍）
   - 示例：
     - `:8 0.6 0.6 0.6 0.6` 表示四个八分音符
     - `0.2:2` 表示二分音符
   - 更新的谱例：
     - `twinkle`（小星星）：添加二分音符标记 `:2`
     - `ode`（欢乐颂）：添加二分音符标记 `:2`
     - `birthday`（生日快乐歌）：修正八分音符和四分音符混合

2. **修复技巧符号渲染**
   - 问题：滑音、击弦、勾弦等技巧符号未正确显示
   - 原因：技巧标记语法错误，应该放在第一个音符上
   - AlphaTab v1.3.1 正确语法：
     - 滑音：`3.6{sl} 5.6` 表示从3品滑到5品
     - 击弦：`3.6{h} 5.6` 表示从3品击弦到5品
     - 勾弦：`5.6{h} 3.6` 表示从5品勾弦到3品（同样用{h}）
     - 推弦：`5.1{b (0 4)}` 表示推高一个全音
     - 闷音：`3.3{pm}` 表示闷音
   - 技巧标记说明：
     - `{h}` = 击弦/勾弦 (Hammer-on/Pull-off)
     - `{sl}` = 连音滑音 (Legato Slide)
     - `{ss}` = 移位滑音 (Shift Slide)
     - `{pm}` = 闷音 (Palm Mute)
     - `{v}` = 颤音 (Vibrato)
     - `{b (值)}` = 推弦 (Bend)
   - 更新的谱例：
     - `slide`：修正滑音语法 `3.6{sl} 5.6`
     - `hammer`：修正击弦语法 `3.6{h} 5.6`
     - `pulloff`：修正勾弦语法 `5.6{h} 3.6`
     - `bend`：新增推弦练习

3. **完善 Scores 注释**
   - 为每个谱例添加详细注释
   - 包含：练习说明、时值说明、技巧说明
   - 便于后续维护和理解

4. **AlphaTex 格式规范**
   - 弦编号：1=第1弦（高音E，最上面），6=第6弦（低音E，最下面）
   - 音符格式：`fret.string`（品格.弦编号）
   - 和弦格式：`(音符 音符)` 同时发声
   - 休止符：`r`

## 2026-05-01 - v1.1.1 AlphaTab API 修复

### Bug修复

1. **修复 AlphaTab v1.3.1 API 调用错误**
   - 问题：`alphaTab.importer.AlphaTexImporter.createFromString` 在 v1.3.1 中不存在
   - 修复：改用 `api.tex(Scores[id], [0])` 方法渲染 AlphaTex 乐谱
   - 设置结构更新为 v1.3.1 嵌套格式（core/display/player）
   - `staves: 'tab'` → `display.staveProfile: 'Tab'`

2. **修复 AlphaTex 格式错误：移除不支持的 `\ts` 命令**
   - 问题：AlphaTab v1.3.1 的 AlphaTex 解析器不支持 `\ts`（拍号）命令
   - 解析器仅识别：title, subtitle, artist, album, words, music, copyright, tempo
   - 修复：从所有 13 个 Scores 中移除 `\ts` 行，默认拍号为 4/4

3. **修复 AlphaTex 音符格式错误：string.fret → fret.string**
   - 问题：AlphaTab v1.3.1 的 AlphaTex 音符格式为 `fret.string`（品格.弦编号），
     弦编号为 1-based（1=第1弦高音E，显示在最上面），
     技巧使用花括号 `{s}`/`{h}`/`{p}`
   - 旧格式为 `string.fret`（弦编号.品格），0-based（0=第1弦），技巧使用后缀 `s`/`h`/`p`
   - 修复：添加自动转换函数，在 Scores 定义后将所有乐谱转为 v1.3.1 格式
   - 转换规则：`S.F` → `F.(S+1)`，技巧后缀 `s`→`{s}`, `h`→`{h}`, `p`→`{p}`

4. **添加 Bravura 音乐字体文件**
   - AlphaTab 需要 Bravura 字体渲染乐谱符号
   - 下载字体文件到 `js/font/` 目录：Bravura.woff, Bravura.otf, Bravura.eot, Bravura.svg

5. **添加本地 HTTP 服务器**
   - 问题：`file://` 协议有 CORS 限制，无法加载字体和 WebAudio Worker
   - 修复：添加 `server.js`，使用 `node server.js` 启动本地服务器
   - 访问地址：http://localhost:9999/

6. **添加 SoundFont 音色库实现播放功能**
   - 问题：AlphaTab 需要 SoundFont 音色库才能播放音频
   - 下载 sonivox.sf2（约1.3MB）到 `js/` 目录
   - 配置 `player.soundFont: 'js/sonivox.sf2'`
   - 现在点击播放按钮可以听到吉他音色
